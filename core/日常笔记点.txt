========Ebean
布尔类型：实体中不需要再为 Boolean 类型的变量指定 @Column(length = 1)，默认就是1，否则会出现生成的创建脚本 tiny(1)(1) 的mysql语法错误

小数类型：@Column(precision = 总位数, scale = 小数位数)，例：1.11对应“@Column(precision = 3, scale = 2)”实际就是限定了确切的整数位和小数位
For float(M,D), double(M,D) or decimal(M,D), M must be >= D (column '列名').
1、这是在为表新增一个字段用于存[0.01, 0.99]这样范围的数准备的，字段就写成了decimal(0, 2)，结果就报了上面的错
2、而实际decimal(M, D)的意思是，整数支持“M-D”位，小数支持“D”位，所以要求“M must be >= D”

========Jpa
通过xxxRepository.getOne(id)方法查询：
1、如果数据库没有没有这个id对应的数据，会抛出非中断异常，但返回的实体对象不为null
2、并在通过该实体获取实体属性时，抛出中断异常，异常信息为没有找到
3、所以所有查询应该用
Optional<T> XxxOptinal = Optional<T> findById(id); 
boolean flag = XxxOptinal.isPresent();
flag false没查到、flag true查到了

----orphanRemoval（默认false）
--单向关系下：A{List<B>}的删除A（前提：单向关系一定得有级联保存关系）
--双向关系下：（前提：B的@ManyToOne的optional属性为true - 允许为空）
额...测试的现象是一样的

orphanRemoval false：删除失败

orphanRemoval true：删除B -> 删除A

CascadeType.REMOVE + orphanRemoval false：删除B -> 删除A

CascadeType.REMOVE + orphanRemoval true：删除B -> 删除A


----Refresh
假如有一条数据，只含有name（"name"）和gender（"male"）
1、A用户取出来在进行修改操作(修改name为“A”),正在A修改的过程中(未提交表单)
2、B用户也对这条数据进行修改操作(修改sex为female),B先将性别修改后提交数据库...
3、接着A用户也提交表单,但是,此时在entityManager中的持久化实体的性别为male,没有更新为B用户修改成的female,所以此时执行一次Refresh操作,就会将该实体更新为数据库中的最新记录,然后再进行提交
4、做级联的时候就会将关联的实体的也获取最新的然后在更新,前提是要执行Refresh操作,CasCadeType.Refresh才会生效
